## What this page is for

This page explains how to **develop** against Morpheum policy and API documentation: editing and previewing the docs locally, and using the **official terminal** for agent and automation workflows. It is for:

- **Developers** contributing to policy docs, API reference, or listing requirements.
- **Agents** that need to validate or update docs content programmatically, or run tests in non-interactive (CLI) mode.
- **Reviewers** who want to check formatting, links, and structure before merge.

The docs site is built with **Mintlify**; you use the Mintlify CLI to serve a local preview. No Morpheum node or API keys are required for editing and previewing. For agent and automation use, the **official terminal** supports both an interactive TUI and a **non-TUI (console) mode** that behaves like a CLI.

<Info>
  **Prerequisites**:
  - Node.js version 19 or higher (for Mintlify docs preview)
  - A clone of this docs repo with a `docs.json` file at the root
  - For terminal usage: Go toolchain and access to the official terminal repository
</Info>

## Official terminal for agent use

Morpheum provides an **official terminal** for both interactive use and **agent use**. It is designed so agents can drive it like a CLI: scriptable, non-interactive, with clear exit status and test categories.

### TUI vs non-TUI (console) mode

- **TUI mode** — Interactive terminal UI: address list, panels (overview, orders, positions, buckets, orderbook, charts, auth, logs, trading, autonomous, endpoints), focus management, and configurable key mappings. Use this when operating the terminal manually.
- **Non-TUI (console) mode** — No interactive UI. The terminal runs as a **CLI**: it parses flags, runs a chosen test category or EIP-712 auth test, and exits with a status. Use this for **agents**, CI, and automation.

### Running in console (agent) mode

You can run the terminal in console mode via a script or the binary.

**Using the console script:**

```bash
# Run simulation tests
bash scripts/run_console.sh simulation

# Run authentication tests with verbose logging
bash scripts/run_console.sh --verbose auth

# Run all tests with a timeout
bash scripts/run_console.sh --timeout 10m all

# EIP-712 authentication test only
bash scripts/run_console.sh --test-auth

# Rebuild and run performance tests
bash scripts/run_console.sh --rebuild performance
```

**Using the binary directly:**

```bash
# Build for console mode (from the terminal repo)
make build-local

# Run by category
./bin/terminal -category simulation -verbose
./bin/terminal -category auth -timeout 10m
./bin/terminal -category all
```

### Test categories (console mode)

| Category      | Description |
|---------------|-------------|
| `simulation`  | Creates multiple addresses and collects bucket info. |
| `auth`        | Creates an address, runs authentication, collects bucket info. |
| `trading`     | Trading system tests. |
| `performance` | Performance benchmarks. |
| `all`         | Runs all categories and aggregates results. |

Results are structured (e.g. category, success, counts, duration) and the process exits with a status suitable for agents and scripts.

### Terminal documentation (TUI, keys, modals)

For interactive use and implementation details, see the terminal repository docs:

- **Key mapping** — Page navigation keys (e.g. 1–7, backtick, x, z, e) and application keys (Ctrl+C, Tab, Enter, Escape); configurable via `key_mappings` in config.
- **Focus and panels** — FocusKeyManager, panel registration, Tab cycling (e.g. user list → menu → content → user list).
- **Modals** — Confirmation, error, success, info; keyboard shortcuts (y/n, o, c, s, d) when a modal is open.
- **API and best practices** — TUI API reference, panel interfaces, thread-safe updates (`QueueUpdateDraw`), and layout (Grid).

Use **console mode** when your workflow is agent- or CLI-driven; use **TUI mode** when you need the interactive terminal.

## Edit and preview the docs

To edit and preview the Morpheum policy and API documentation on your machine, use the Mintlify CLI.

### Install and run the preview

<Steps>
<Step title="Install the Mintlify CLI">

```bash
npm i -g mint
```
</Step>

<Step title="Start the local preview">

Go to the directory that contains `docs.json` and run:

```bash
mint dev
```

The docs will be available at `http://localhost:3000`.
</Step>
</Steps>

### Custom ports

By default, Mintlify uses port 3000. To use another port, use the `--port` flag:

```bash
mint dev --port 3333
```

If port 3000 is in use, Mintlify will try the next available port and print a message like:

```md
Port 3000 is already in use. Trying 3001 instead.
```

### Mintlify versions

Each Mintlify CLI release matches a specific docs runtime. If your local preview looks different from production, update the CLI:

```bash
npm mint update
```

### Validating links

To find broken links in the documentation, run:

```bash
mint broken-links
```

### Code formatting

Use IDE extensions for MDX and formatting. For VS Code:

- [MDX extension](https://marketplace.visualstudio.com/items?itemName=unifiedjs.vscode-mdx) for syntax highlighting
- [Prettier](https://marketplace.visualstudio.com/items?itemName=esbenp.prettier-vscode) for consistent formatting

### Troubleshooting

<AccordionGroup>
  <Accordion title='Error: Could not load the "sharp" module using the darwin-arm64 runtime'>

    This often comes from an old Node version. Try:
    1. Remove the CLI: `npm remove -g mint`
    2. Upgrade to Node v19 or higher.
    3. Reinstall: `npm i -g mint`
  </Accordion>

  <Accordion title="Unknown or persistent error">
  
    Delete the Mintlify cache: remove the `~/.mintlify` folder, then run `mint dev` again.
  </Accordion>
</AccordionGroup>

For CLI changes and releases, see the [Mintlify CLI changelog](https://www.npmjs.com/package/mintlify?activeTab=versions).
